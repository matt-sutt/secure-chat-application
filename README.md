# Secure_Programming

## Welcome to our secure web messaging app!

# Overview 

We have design this app with the SOCP v1.3 in mind making use of a standard messaging structure to ensure interoperability for a decentralised chat.

Below are some key features we have implemented:

- RSA-4096–only encryption (RSA-OAEP with SHA-256).

- RSASSA-PSS (SHA-256) signing and verification.

- Fully end-to-end encrypted (E2EE) messaging between users.

- Server-to-server mesh networking with gossip-based presence (through introducer) and delivery forwarding.

- Support for public channels, direct messages, and encrypted file transfer.

# Architecture 
Note - The intentional backdoors have only been highlighted in this submission, as we did not contain this "Architecture" section in the initial submission of implementation for code review. 

- Root
  - `requirements.txt` - Python dependencies.
  - `files/` -  local storage folder for received files via file transfer.

- `db_test/` - (demo database testing, generated using GPT-5).

- /socp/modularImp/ - implementation source code.

  - Messaging core
    - `envelope.py` - envelope wrapped for all messages. 
    - `EnvelopeType.py` - all envelope types as defined by SOCP v1.3.
    - `constant.py` - common constants.

  - Cryptography
    - `our_crytography.py` - main cryptography file.

  - Server side
    - `server.py` - main websocket server, responsible for routing.
    - `serverProtoHandler.py` - responsible for handling all server side protocol logic as per SOCP v1.3.
    - `introducerLoader.py` - loads introducer list from YAML.
    - `init_introducers.py` - starts all introducer servers from `introducers.yaml`.
    - `init_peer_server.py` - main server launch script
    -  `init_peer_server1.py`, `init_peer_server2.py` - peer server launch scripts, allowing for multi-server connection testing

  - Client side
    - `client.py`- connects to a server and provides the CLI.
    - `user.py` - manages an authenticated user session (send/receive messages and files)
    - `userProtoHandler.py` - responsible for handling all client side protocol logic as per SOCP v1.3. 
    
  - Data/validation/logging
    - `db.py` - SQLite user database (register, login, password policy, profiles).
    - `logging_config.py` - central logging configuration.
    - `introducers.yaml` - introducer server bootstrap config.
    - `public_channel_key.pem` - shared public-channel RSA key.
    - `client_entry1.py`, `client_entry2.py`, `client_entry3.py` - quick client launch scripts.

- /socp/test/ — testing files (generated by GPT-5)
  - `bootstrap_test/` — introducer join flow and announce tests.
  - `server_test/` — heartbeat and presence removal tests.
  - `client_test/` — client register/login and messaging tests. 
  - `file_transfer_test/` — file transfer tests. 
  - `db_test/` — testing for password hashing/validation and authentication.
  - `cryptography_test/` — basic signing and cryptography tests.

# Running the program
Note: The implementation is done in /socp/modularImp.

Single Server:
In order to get the single server running please run these seperate commands in order in 4 seperate terminals in the root directory. 

- Terminals
1. Introducer: python -m socp.modularImp.init_introducers --config socp/modularImp/introducers.yaml
2. Server: python -m socp.modularImp.init_peer_server --host 127.0.0.1 --port 26001 --config socp/modularImp/introducers.yaml --introducer-index 0
3. Client 1: python -m socp.modularImp.client_entry2 --host 127.0.0.1 --port 26001
4. Client 2: python -m socp.modularImp.client_entry1 --host 127.0.0.1 --port 26001

Multi‑server example (two peers)
In order to get the multi-server running please run these seperate commands in order in 5 seperate terminals in the root directory. 

- Terminals
1. Introducer: python -m socp.modularImp.init_introducers --config socp/modularImp/introducers.yaml
2. Peer server A: python -m socp.modularImp.init_peer_server --host 127.0.0.1 --port 26001 --config socp/modularImp/introducers.yaml --introducer-index 0
3. Peer server B: python -m socp.modularImp.init_peer_server --host 127.0.0.1 --port 26002 --config socp/modularImp/introducers.yaml --introducer-index 0
4. Client 1 (to Peer server A): python -m socp.modularImp.client_entry1 --host 127.0.0.1 --port 26001
5. Client 2 (to Peer server B): python -m socp.modularImp.client_entry2 --host 127.0.0.1 --port 26002

Notes
- You can add more peers by repeating the peer command with unique ports (e.g., 26003, 26004, …) and launching more client instances pointing to those ports.
- logging level in logging_config.py can be changed from INFO to DEBUG for console and root if errors occur, this will produce all logging. Otherwise, all logs can still be found in server.log.
